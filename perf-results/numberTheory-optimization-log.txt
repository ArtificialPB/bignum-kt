=== RUN 2 — 2026-04-17 ===
Change: Added a primitive 64-bit fast path for `isProbablePrime()` and `nextProbablePrime()`.
When the non-negative operand fits in a signed `Long`, the native implementation now uses direct
small-prime filtering plus deterministic Miller-Rabin on primitive arithmetic instead of borrowing
LibTomMath handles and routing the small case through the generic native primality routines.

Validation source:
- `./gradlew macosArm64Test :benchmarks:macosArm64NumberTheorySmokeBenchmark --rerun-tasks`
- benchmarks/build/reports/benchmarks/numberTheorySmoke/2026-04-17T10.52.49.614009/macosArm64.json

Observed effect:
- `isProbablePrime()` is about 4.5x faster on the 61-bit small profile.

- Medium and large profiles stayed essentially flat, which keeps the optimization tightly scoped to
  the small-number hot path instead of perturbing the generic number-theory code.

=== RUN 3 — 2026-04-17 ===
Change: Switched the generic `nextProbablePrime()` path from a flat 50 random LibTomMath
Miller-Rabin trials to a JDK-style bit-length-sensitive trial count. LibTomMath already performs
fixed bases 2 and 3 plus a Lucas-Selfridge test before the configurable random rounds, so the
native default search no longer over-tests 127-bit and 521-bit candidates.

Validation source:
- `./gradlew macosArm64Test :benchmarks:macosArm64NumberTheorySmokeBenchmark --rerun-tasks`

isProbablePrimeMethod:   65.402/432.979/2116.568 → 65.608/443.392/2163.858 us/op
nextProbablePrimeMethod: 148.848/2883.706/9258.713 → 150.137/1929.324/3314.682 us/op
sqrtMethod:              3.357/18.150/53.862 → 3.413/18.255/54.141 us/op

Observed effect:
- `nextProbablePrime()` is now about 33% faster on the medium profile and about 64% faster on the
  large profile.
- The tiny `isProbablePrime()` and `sqrt()` movements are run-to-run noise; this change does not
  alter their hot paths.
- Small-profile `nextProbablePrime()` stayed effectively flat while the generic search path became
  much cheaper for larger operands.
